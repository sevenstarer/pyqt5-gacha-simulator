# PyQt5 抽卡模拟游戏（课程大作业）
基于Python+PyQt5开发，复刻FGO抽卡概率与软硬保底机制，包含抽卡、图鉴、数据统计、存档重置功能。
所有工作均由组长负责
## 环境依赖
```bash
pip install -r requirements.txt  
```  
## 程序功能介绍  
### 项目简介  
本程序基于 Python3 + PyQt5 图形界面框架开发二次元手游风格抽卡模拟器，参考 FGO 召唤机制设计四档卡牌概率、十连保底机制，搭配专属召唤动画弹窗、本地数据持久化、卡牌收集图鉴、抽卡数据统计功能，完整实现可视化抽卡交互程序。
程序使用 Git 完成版本管理，所有代码托管至 GitHub 远程仓库，代码分层模块化，具备完善容错处理，可直接在 Windows 环境一键运行。  
### 核心完整功能  
#### 基础抽卡功能  
1. 支持单抽、十连两种抽卡模式；  
2. 内置四品级卡牌池：普通、稀有、史诗、传说（五星），配置固定抽取概率；  
3. 十连保底规则：十连必出至少一张稀有及以上卡牌，模拟手游保底逻辑；  
4. 实时打印每一次抽卡结果至界面文本框，留存抽卡记录。  
#### 可视化多标签界面  
采用QTabWidget实现三页面切换：  
1. 抽卡页面：展示剩余抽卡次数、抽卡记录、单抽 / 十连 / 重置数据按钮；  
2. 卡牌图鉴页面：自动展示所有已抽取收集的卡牌，按品级区分；  
3. 数据统计页面：统计四类卡牌抽取总数量，直观展示概率分布。  
#### 自定义加载抽卡动画  
1. 自定义召唤背景图加载渲染，支持 jpg 图片填充弹窗；  
2. 分级差异化动画：普通卡牌仅基础淡入效果，史诗卡牌缩放回弹动画；  
3. 传说（五星）专属特效：全屏金色渐变光圈、随机白色星光粒子、卡牌放大回弹动画；  
4. 模态弹窗自动关闭，十连采用延时定时器逐张播放动画，模拟连续召唤流程；  
#### 本地数据持久化存储  
1. 使用 JSON 文件保存所有抽卡数据，程序启动自动加载存档；  
2. 存档存储内容：剩余抽卡次数、已收集卡牌集合、各品级卡牌抽取总数；  
3. 一键重置功能：清空本地存档，重置全部抽卡数据。  
## 项目各模块与类设计细节  
### 入口启动模块 main.py  
功能：程序唯一入口，初始化主窗口并启动 Qt 事件循环。  
核心代码逻辑：导入主窗口类，创建应用实例，展示主界面，开启程序循环。  
### 业务逻辑模块 core/draw_logic.py  
负责纯抽卡算法与数据读写，与界面完全解耦，包含 2 个核心功能单元：  
#### 抽卡概率算法单元  
内置卡牌池字典，存储各品级卡牌名称；  
get_single_card()：生成单张卡牌，通过随机数匹配对应品级；  
get_ten_cards()：生成十连卡牌列表，执行保底校验，无稀有卡牌则自动替换一张；  
#### 本地存档读写单元  
load_save()：读取 data/save.json，文件不存在则初始化空白存档；  
update_save()：抽卡完成后更新剩余次数、收集卡牌、品级统计并写入 JSON。  
### 主界面交互模块 ui/main_ui.py  
主窗口类 MainWindow(QWidget)，页面交互与控件绑定：  
1. 界面布局：QTabWidget 三标签页、QLabel 文字展示、QTextBrowser 记录框、QPushButton 功能按钮；  
2. 槽函数绑定：  
on_single()：调用单抽逻辑，创建动画弹窗并模态展示；  
on_ten()：循环执行十连抽，通过 QTimer 延时逐张弹出动画；  
reset_data()：清空本地存档，刷新页面所有数据；  
3. 页面刷新函数：抽卡完成后自动更新剩余次数、图鉴列表、统计数据。  
### 召唤动画弹窗模块 ui/draw_anim_dialog.py  
弹窗类 DrawAnimDialog(QDialog)，实现背景渲染与分级动画，核心设计细节：  
1. 构造函数 __init__  
2. 绘图重写 paintEvent(event):重写 Qt 绘图事件，优先绘制背景图，再叠加传说卡牌金光、星光粒子特效；仅当卡牌品级为传说时，渲染全屏金色光圈与随机星光；  
3. 自动关闭逻辑：动画播放 3 秒后自动关闭弹窗。

### 开发过程遇到的问题与解决反思  
1. 动画弹窗黑屏问题  
问题：加载背景图片后窗口依旧纯黑，终端持续输出空 Pixmap 警告；  
反思：最初忽略paintEvent绘图事件，仅加载图片但未执行绘制；同时存在文件名拼写错误、强制黑色样式覆盖图片的问题。后续优化：增加图片加载调试打印、完善绘图判空逻辑、删除遮挡背景的黑色样式，彻底解决黑屏。   
2. Qt 运行时父类初始化报错  
问题：窗口控件操作写在super().__init__()之前，程序直接崩溃；  
反思：不熟悉 Qt 窗口类初始化规则，所有窗口操作必须在父类构造执行完成后调用，后续规范代码书写顺序，规避同类错误  
3. 十连动画弹窗重叠卡顿  
问题：同步循环创建弹窗，多张窗口同时弹出界面卡顿；  
反思：未使用异步延时，循环同步阻塞界面，改用QTimer设置递增延时，逐张串行播放动画，优化交互流畅度。
